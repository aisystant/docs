#### §3.04 MCP: связующая ткань IWE

**В одном предложении:** MCP — это не «техническая деталь», а нервная система вашего экзокортекса: способ, которым Pack, агенты, протоколы и внешние инструменты обмениваются контекстом, не заставляя вас играть роль курьера между ними.

---

**Сигнатура понятий:**

- **Задаётся:** MCP (Model Context Protocol) — протокол передачи контекста между компонентами IWE; правила, по которым агент получает доступ к Pack, протокол получает доступ к агенту, а внешний инструмент — к протоколу
- **Задаётся:** Контекстный протокол — соглашение о том, какие данные передаются, в каком формате и с какими правами; без протокола каждое соединение требует ручной настройки и быстро ломается
- **Противопоставлен:** «интеграция через копипаст» — ручной перенос данных между инструментами; работает на малом масштабе, ломается при росте системы
- **Используется:** IWE (§3.01) — MCP — один из компонентов третьего уровня (интерфейс)
- **Используется:** Агенты (§3.03) — агенты работают через MCP; без протокола агент не получает контекст Pack

---

**Объяснение**

> Если Pack — это стеллажи, агенты — станки, то MCP — это транспортная система мастерской: конвейеры, лифты, тележки, по которым материал движется от хранилища к обработке и обратно. Без транспорта станки стоят, а стеллажи захламляются.

**Почему без MCP система ломается.** Большинство людей сталкиваются с проблемой раньше, чем узнают её название. Заметки в одном приложении, задачи в другом, календарь в третьем, ИИ в четвёртом. Информация между ними передаётся вручную: скопировал, вставил, забыл обновить. Это работает, пока система маленькая. При росте ручная передача становится главным узким местом.

MCP решает эту проблему на архитектурном уровне: он задаёт правила, по которым компоненты обмениваются данными автоматически. Не «я скопировал заметку в чат», а «агент прочитал Pack и предложил действие». Не «я вспомнил, что надо сделать», а «протокол проверил состояние и напомнил».

**Что MCP передаёт.** Контекст — это не просто «данные». Это данные плюс их значение плюс правила использования. MCP передаёт:
- **Содержание:** что именно лежит в Pack — заметки, принципы, решения.
- **Структуру:** как это организовано — категории, связи, версии.
- **Правила:** что с этим можно делать — читать, изменять, агрегировать, удалять.

Без структуры агент получает «кучу текста» и теряется. Без правил агент может изменить то, что не должен. MCP защищает и от того, и от другого.

**MCP и автоматизация.** Когда MCP работает, система начинает действовать сама: агенты проверяют Pack по расписанию, протоколы запускаются по триггерам, внешние события (письмо, коммит, заметка) инициируют цепочки действий. Человек перестаёт быть курьером и становится оператором: наблюдает, корректирует, решает.

Это не значит, что человек выбывает из процесса. Это значит, что его роль меняется с «делаю всё руками» на «контролирую систему». Точно так же, как водитель автомобиля не толкает машину, а управляет ею.

**Типичная ошибка.** «У меня и так всё работает, зачем протокол?» — эта фраза обычно произносится человеком, у которого 3–4 инструмента и 10–15 ручных операций в день. Он не замечает этих операций, потому что они стали привычкой. Но привычка — не система. При увеличении масштаба ручные операции становятся первым местом, где всё разваливается.

**Степени мастерства:**

| Степень | Что происходит | Критерий перехода |
|---------|----------------|-------------------|
| Объяснение | Можете объяснить, чем MCP отличается от «интеграции через копипаст», и назвать три вещи, которые передаёт протокол | — |
| Умение | Есть хотя бы одна автоматическая связь между двумя компонентами IWE (например, агент читает Pack без вашего ручного копирования) | Есть запись о практике |
| Навык | Система обменивается контекстом между 3+ компонентами; вы редко копируете вручную | Регулярная работа без ручной передачи |
| Мастерство | Проектируете связку компонентов для другого человека | Есть кейс передачи |

---

**Проверка себя:**

- **Индикатор понимания:** вы можете нарисовать схему своего IWE и показать, какие стрелки между компонентами идут автоматически, а какие — вручную.
- **Индикатор поведения:** при появлении новой задачи вы сначала думаете «как это встроить в поток данных», а не «куда записать».
- **Подсказка при застревании:** если «всё разбросано по разным местам» — не начинайте с уборки. Начните с одной автоматической связи: пусть одно приложение забирает данные из другого без вашего участия.

---

**На практике:**

- Нарисуйте схему текущих инструментов: круги — приложения, стрелки — потоки данных. Отметьте стрелки, которые вы проводите руками.
- Выберите одну ручную стрелку, которую можно автоматизировать за 15 минут.
- Автоматизируйте. Это начало MCP в вашей системе.
