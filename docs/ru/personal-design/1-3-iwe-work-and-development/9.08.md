#### §9.08 Список понятий раздела

**Время:** 20 мин чтение + 30 мин

**В одном предложении:** Раздел 9 ввёл семь взаимосвязанных понятий, которые вместе формируют архитектуру агентной экосистемы: от навигации по траектории до маркировки достоверности знаний.

Ниже — сводная таблица всех ключевых понятий раздела. Проверьте: можете ли объяснить каждое понятие своими словами и привести пример из своей практики? Если нет — вернитесь к соответствующему подразделу.

| Понятие | Определение | Подраздел |
|---------|-------------|-----------|
| **RouteGuide** (Проводник по траектории) | Механизм навигации пилота по траектории развития, реализованный через Pack, агентов-Навигаторов и протоколы; знает текущую ступень, цель и следующие шаги | §9.01 |
| **Траектория** | Упорядоченный путь развития пилота через ступени мастерства с чёткими критериями входа, пребывания и выхода | §9.01 |
| **Ступень** | Дискретный уровень мастерства с чёткими критериями, зафиксированными в Pack | §9.01 |
| **FSM** (Конечный автомат) | Модель управления состояниями, в которой система находится в одном из конечного числа состояний, а переходы инициируются событиями и ограничены условиями | §9.02 |
| **Состояние** | Дискретный режим функционирования агента, РП или протокола с чётким набором допустимых действий | §9.02 |
| **Переход** | Смена состояния, triggered by событие и guarded by условие | §9.02 |
| **Эпистемический граф** | Граф знаний, где вершины — понятия, а рёбра — эпистемические связи (обоснование, зависимость, противоречие) | §9.03 |
| **Вершина** | Понятие, узел или сущность в Pack, имеющее чёткое определение и границы | §9.03 |
| **Ребро** | Эпистемическое отношение между вершинами: обоснование, зависимость или противоречие | §9.03 |
| **Валидация** | Проверка, что решение или выходные данные соответствуют Pack, FPF и критериям приёмки | §9.04 |
| **Трассировка** | Способность проследить любое решение до источника: данных, предпосылок и агентов | §9.04 |
| **Решение** | Зафиксированный выбор среди альтернатив с прослеживаемой аргументацией | §9.04 |
| **Экономика вклада** | Система учёта и поощрения вклада пилота, построенная на принципе доказуемости результата | §9.05 |
| **Proof-of-Impact** (PoI) | Фиксация результата, который может быть проверен другими: воспроизведён, измерен или прослежен до источника | §9.05 |
| **Вестинг** | Накопление прав, репутации или доступа через доказуемый вклад за определённый период | §9.05 |
| **Fan-out** (Веерный поиск) | Распределение запроса к нескольким агентам или источникам одновременно с последующей агрегацией результатов | §9.06 |
| **LLM Proxy** (Посредник модели) | Посредник, маршрутизирующий запросы к оптимальной языковой модели на основе типа задачи, latency и стоимости | §9.06 |
| **Маршрутизация** | Процесс выбора наилучшего пути исполнения запроса в распределённой экосистеме | §9.06 |
| **Epistemic status** | Маркировка уровня уверенности в утверждении: доказано, гипотеза, предположение, анекдот | §9.07 |
| **Гипотеза** | Проверяемое предсказание с определёнными критериями подтверждения или опровержения | §9.07 |
| **Факт** | Утверждение, верифицированное против независимой эвиденции с прослеживаемым происхождением | §9.07 |

**Метод — минимальный шаг.** Практика «Проверка понятий» (30 мин):

1. Закройте таблицу. Возьмите чистый лист. (1 мин)
2. Запишите все понятия, которые помните. (10 мин)
3. Для каждого напишите определение своими словами без заглядывания в текст. (12 мин)
4. Откройте таблицу и сравните. Отметьте: какие понятия ясны, какие — требуют возврата к подразделу. (7 мин)

**Проверка себя.**

- Могу объяснить каждое понятие из таблицы своими словами за 30 секунд
- Могу провести эпистемическое ребро между понятиями из разных подразделов
- Могу назвать три пары понятий, которые конфликтуют или дополняют друг друга
