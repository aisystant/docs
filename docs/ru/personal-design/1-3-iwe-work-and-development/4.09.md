#### §4.09 Упражнения на понятия

**Упражнение 1. Сопоставление ролей (10 мин)**

Перед вами ситуации. Определите, какую роль агента следует привлечь. Объясните, почему.

1. Нужно написать спецификацию API по шаблону
2. Нужно понять, почему метрики упали за последнюю неделю
3. Нужно проверить, соответствует ли план недели фактическим действиям
4. Нужно скоординировать работу трёх агентов над одним проектом
5. Нужно найти ошибку в коде, который вчера написал агент

**Ответы:**
1. Портной — создаёт артефакты по шаблону
2. Диагност — анализирует данные и находит причины
3. Навигатор — контролирует соответствие плана действиям
4. Оркестратор — координирует поток между агентами
5. Диагност (peer-review) — проверяет артефакт другого агента

**Упражнение 2. Диагностика автопилота (10 мин)**

Прочтите сценарий. Определите: автопилот или экзоскелет? Объясните.

*Сценарий:* Пилот использует агента для написания еженедельных отчётов. Агент пишет, пилот читает и отправляет. Пилот не может написать отчёт без агента: не знает структуру, не помнит метрики, не уверен в формулировках.

*Вопрос:* Автопилот или экзоскелет? Какие три действия вернут пилота к экзоскелету?

*Ответ:* Автопилот. Пилот делегировал не наборание текста, а мышление. Действия: 1) Написать отчёт без агента, пусть плохо. 2) Изучить структуру и метрики, зафиксировать в Pack. 3) Использовать агента как редактора, не автора.

**Упражнение 3. Проверка изоляции (20 мин)**

1. Вспомните три последние сессии с агентами. Для каждой: был ли контекст изолирован?
2. Найдите один случай, когда контекст одной задачи мог повлиять на другую.
3. Составьте правило изоляции для вашей IWE: что агент видит, что не видит, как переключается контекст.
4. Зафиксируйте правило в Pack.
