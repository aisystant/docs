#### §5.05 Множественные агенты

**Время:** 45 мин чтение + 30 мин

**В одном предложении:** Человек думает, что один хороший агент делает всё, а на самом деле один агент — это одна пара рук, и если вы хотите работать быстрее, вам нужно не более мощный агент, а правильно распределённая команда агентов, где каждый делает своё, пока другие делают своё.

**Мем, который снимается.** «У меня один агент, он справляется со всем. Зачем мне несколько? Это сложно, дорого, запутанно.» На самом деле один агент не справляется со всем. Он переключается между задачами, теряя контекст. Он не может одновременно писать код и анализировать метрики. Он не может думать о стратегии, пока пишет документ. Множественные агенты — это не роскошь, это мультипликатор: пока один агент работает над сложным РП, другой готовит контекст для следующего, третий проверяет результаты первого. Время пилота используется параллельно, а не последовательно.

**Определение из источника.** Множественные агенты — использование двух и более агентов в разных ролях для параллельного выполнения задач. Распределение — разделение труда между агентами: каждый агент выполняет роль, соответствующую его сильным сторонам. Сильная модель работает над сложным РП, требующим глубокого анализа. Слабая модель работает над рутиной: форматированием, проверкой, подготовкой контекста. Context isolation — обязательное условие: агенты не смешивают контексты, не перезаписывают результаты друг друга, не путают задачи. Мультиагентность требует координации: пилот или Оркестратор распределяет задачи, собирает результаты, разрешает конфликты.

**Развитие мысли.** Множественные агенты работают по принципу конвейера. Агент A готовит черновик. Пока A работает, пилот передаёт контекст агенту B для подготовки следующей задачи. Когда A заканчивает, B уже готов начать. Пилот не ждёт — он координирует. Этот принцип особенно эффективен при работе с разными моделями: одна модель быстрая, но поверхностная — для рутины. Другая медленная, но глубокая — для анализа. Третья — для верификации. Пока глубокая модель думает, быстрая выполняет десять мелких задач. Это не теория — это практика: время пилота ограничено, агентов можно запускать параллельно. Главное ограничение — не количество агентов, а качество координации. Без чёткого распределения ролей множественные агенты создают хаос: каждый делает что-то своё, результаты не согласуются, пилот тратит время на разбор. С распределением — синергия: сумма больше частей.

**Метод — минимальный шаг.** Практика «План на день с двумя агентами" (30 мин):

1. Выберите завтрашний день. Определите 2–4 РП. (5 мин)
2. Распределите РП между агентами: какой агент выполняет какую роль? Какие РП можно параллелить? (15 мин)
3. Определите очерёдность: что начинается первым, что зависит от чего. Зафиксируйте план. (10 мин)

**Пример из жизни.** Предприниматель имел один сложный РП — анализ рынка для нового продукта — и десять мелких: ответы на письма, обновление сайта, подготовка презентации. С одним агентом он делал всё последовательно: анализ занял три часа, мелкие задачи — ещё два. С множественными агентами: глубокая модель (Claude) работала над анализом рынка. Пока она думала, быстрая модель (Kimi) отвечала на письма и обновляла сайт. Когда Claude закончил анализ, Kimi уже подготовил черновик презентации на основе результатов. Общее время сократилось с пяти часов до трёх. Но главное — пилот не переключался между контекстами: он работал с Claude над анализом, пока Kimi делал рутину.

**Типичная ошибка.** «Я запущу пять агентов, и всё будет делаться само.» Пять агентов без координации — это пять источников хаоса. Начните с двух. Выстройте распределение. Добавляйте третьего только когда с двумя стабильно. Другая ошибка: «У меня нет времени на координацию нескольких агентов.» Время на координацию — инвестиция. Оно окупается за счёт параллельности. Но только если координация есть. Без неё — чистые издержки.

**Степени мастерства:**

1. Объяснение. Могу объяснить, зачем нужны множественные агенты, и привести пример распределения ролей. Критерий перехода: составил план использования двух агентов.
2. Умение. Регулярно использую два агента в разных ролях: один для сложных, другой для рутинных задач. Критерий перехода: две недели стабильного использования.
3. Навык. Системно использую множественных агентов: планирование, координация, context isolation, сбор результатов. Критерий перехода: коллега заметила, что «вы делаете в два раза больше за то же время".
4. Мастерство. Проектирую мультиагентную систему для команды: роли, протоколы, инструменты координации. Критерий перехода: другой человект использовал вашу систему множественных агентов.

**Проверка себя.**

- Могу назвать две роли, которые выполняют разные агенты в моей IWE
- Составлял план использования нескольких агентов на один день
- Использую context isolation: агенты не смешивают контексты
- Могу объяснить, какие задачи параллелятся, а какие — последовательны
- Проверял результаты работы каждого агента перед сбором в единый продукт

**На практике.** Завтра выберите одно рутинное РП и одно сложное. Назначьте рутинное быстрому агенту, сложное — глубокому. Запустите параллельно. Сравните время с последовательным выполнением. Это и есть множественные агенты.
