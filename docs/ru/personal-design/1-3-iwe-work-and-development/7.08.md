#### §7.08 Список понятий раздела

**Время:** 20 мин чтение + 15 мин

**В одном предложении:** Раздел 7 вводит понятия развития IWE: от перехода от использования к созданию до управляемой эволюции среды через протоколы, подключения, извлечение знания и регулярное техническое обслуживание.

| Понятие | Определение | Подраздел |
|---------|-------------|-----------|
| M5 (Созидание) | Уровень мастерства создания IWE: проектирование Pack, протоколов, MCP-серверов | §7.01 |
| M2 (Управление) | Уровень мастерства использования существующей IWE: следование протоколам, настройка параметров | §7.01 |
| Переход от M2 к M5 | Смена качества работы пилота: от потребителя готовых решений к создателю новых | §7.01 |
| Протокол | Формализованная последовательность действий с критериями входа, проверками и выхода | §7.02 |
| Формализация | Процесс абстракции общих шагов из конкретных случаев и фиксации в пригодном для повторного использования виде | §7.02 |
| Тестирование протокола | Проверка работоспособности формализации на трёх и более независимых случаях | §7.02 |
| MCP-дизайн | Проектирование интерфейса взаимодействия агента с внешними инструментами через Model Context Protocol | §7.03 |
| Параметры вызова | Аргументы, которые агент передаёт инструменту для выполнения задачи | §7.03 |
| Обработка ошибок | Стратегия поведения агента при сбое подключения: fallback, логирование, уведомление пилота | §7.03 |
| Knowledge Extraction | Извлечение и маршрутизация знания на рубеже работы: паттернов, решений, различений | §7.04 |
| Паттерн | Повторяющаяся структура в данных или поведении, пригодная для формализации | §7.04 |
| Различение | Уточнение границы между понятиями, ранее воспринимавшимися как одно | §7.04 |
| ТО (Техническое обслуживание) | Регулярное профилактическое обслуживание IWE для предотвращения деградации | §7.05 |
| Аудит | Систематическая проверка состояния компонентов IWE с выдачей вердикта и плана действий | §7.05 |
| Дрейф-контроль | Отслеживание расхождения между текущим состоянием IWE и её целевой архитектурой | §7.05 |
| Context Engineering | Проектирование информационного окружения запроса к агенту для получения качественного ответа | §7.06 |
| Prompt | Формулировка запроса к агенту: что попросить, в какой форме, с какими ограничениями | §7.06 |
| Контекстное окно | Объём информации, который агент может обработать за один запрос; ограниченный ресурс | §7.06 |
| Эволюция IWE | Направленное изменение среды на основе рефлексии периода с сохранением стабильности | §7.07 |
| Инвариант | Элемент IWE, который работает и должен быть сохранён при изменениях | §7.07 |
| Изменение | Одна модификация, вносимая в IWE за период (неделя или месяц) | §7.07 |

**Практика (15 мин):** Пройдитесь по таблице сверху вниз. Для каждого понятия сформулируйте определение своими словами, не подглядывая в текст. Если не получается — вернитесь к соответствующему подразделу. Цель: убедиться, что каждое понятие перешло из текста в понимание.

**См. также:** §7.01 — §7.07 для развёрнутых определений и примеров.
