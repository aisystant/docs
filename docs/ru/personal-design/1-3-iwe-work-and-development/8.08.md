#### §8.08 Список понятий раздела

Сводная таблица всех понятий, введённых в Разделе 8. Используйте её для проверки: можете ли вы объяснить каждое определение своими словами, не заглядывая в текст?

| Понятие | Краткое определение | Введено |
|---------|---------------------|---------|
| **Экзоскелет** | Режим работы, в котором ИИ усиливает сохранённую компетенцию человека: человек решает, машина поддерживает | §8.01 |
| **Замещение** | Режим, в котором пилот делегирует мышление ИИ, постепенно теряя способность выполнять задачу самостоятельно | §8.01 |
| **Тест навыка** | Проверка: «Если ИИ исчезнет прямо сейчас, останется ли у меня навык выполнить эту задачу?» | §8.01 |
| **Объяснение** | Способность сформулировать логику решения своими словами, без опоры на вывод агента | §8.02 |
| **Проверка** | Верификация вывода агента перед принятием решения; не доверие, а контроль | §8.02 |
| **Стоп-лист** | Набор критериев, при выполнении которых работа над РП прерывается независимо от желания пилота | §8.03 |
| **Бюджет** | Лимит ресурсов (время, энергия, деньги), выделенных на РП; исчерпание = стоп | §8.03 |
| **Состояние** | Текущее состояние пилота (bh.stb — стабильность); падение ниже порога = стоп | §8.03 |
| **Алерт** | Сигнал Машины о наступлении или приближении проблемного состояния | §8.04 |
| **Детектор** | Скрипт или агент, распознающий паттерн и инициирующий алерт при его обнаружении | §8.04 |
| **Тренд** | Динамика показателя во времени; рост или спад, видимый при агрегации данных | §8.04 |
| **Авария IWE** | Состояние, при котором Машина или её использование создают диссонанс с намерениями Пилота | §8.05 |
| **Потеря фокуса** | Трансформация мультисессионности в многозадачность внутри одного фокусного слота | §8.05 |
| **Сжигание времени** | Растрата фокусных слотов на взаимодействие с агентами как развлечение, а не как работу | §8.05 |
| **Восстановление** | Процесс возврата IWE в стабильное состояние после аварии по пяти шагам | §8.06 |
| **Превенция** | Изменение в IWE, предотвращающее повторение аварии; работа с причиной, а не следствием | §8.06 |
| **Фиксация** | Запись в Pack того, что случилось, что делали, что изменили; превращение аварии в опыт | §8.06 |
| **Knowledge Gate** | Фильтр на входе знания в Pack: проверяет связь, дублирование, проверенность, контекст | §8.07 |
| **Знание** | Структурированная информация, прошедшая Gate и интегрированная в Pack | §8.07 |
| **Контекст** | Условия, при которых знание применимо: задача, этап, домен; без контекста знание — мёртвый груз | §8.07 |
