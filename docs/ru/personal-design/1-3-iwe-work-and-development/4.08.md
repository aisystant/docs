#### §4.08 Список понятий раздела

| Понятие | Определение | Подраздел |
|---|---|---|
| Навигатор | Роль ориентации: поддерживает траекторию, контролирует соответствие плана действиям | 4.01 |
| Портной | Роль созидания: создаёт артефакты, воплощает замысел пилота | 4.01 |
| Диагност | Роль анализа: оценивает состояние, находит проблемы | 4.01 |
| Оркестратор | Роль координации: управляет потоком между агентами | 4.01 |
| MCP | Model Context Protocol — протокол связи агента с инструментами | 4.02 |
| MCP-сервер | Программный модуль, предоставляющий агенту доступ к инструменту | 4.02 |
| Контур | Граница, внутри которой агент работает с инструментами | 4.02 |
| IntegrationGate | Пятиступенчатый процесс подключения нового агента | 4.03 |
| Context isolation | Принцип изолированного контекста для каждого агента | 4.04 |
| Контекстное загрязнение | Проникновение контекста одной задачи в другую | 4.04 |
| Peer-review | Перекрёстная проверка артефактов между агентами и пилотом | 4.06 |
| Верификация | Проверка результата, а не процесса создания | 4.06 |
| Трассировка | Прослеживание решения до источника | 4.06 |
| Автопилот | Режим, при котором агент действует самостоятельно, пилот одобряет | 4.07 |
| Экзоскелет | Режим, при котором агент усиливает способности пилота | 4.07 |
| Когнитивная независимость | Способность пилота выполнять задачи без ИИ | 4.07 |

**Практика (15 мин):** Пройдитесь по списку. Для каждого понятия скажите вслух или запишите: «Это значит...» — своими словами, без чтения определения. Если застряли — вернитесь к соответствующему подразделу.
