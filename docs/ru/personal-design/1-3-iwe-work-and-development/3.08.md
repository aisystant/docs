#### §3.08 Список понятий раздела

| Понятие | Определение | Подраздел |
|---|---|---|
| Pack (контекст IWE) | Живой документ — карта знаний и практик пилота, обновляемый по ходу работы | 3.01 |
| База знаний | Статическое хранилище: записал — забыл, оно лежит | 3.01 |
| Живой документ | Документ, который меняется по ходу работы и отражает текущее состояние | 3.01 |
| Иерархия знаний | Система уровней: ZP → SPF → FPF → Pack → DS | 3.02 |
| ZP | Законы природы — неизменные основания | 3.02 |
| SPF | Принципы предметной области — устойчивые, но проверяемые | 3.02 |
| FPF | Фундаментальные принципы функционирования — основания, задающие рамки Pack | 3.02, 3.04 |
| DS | Дерево систем — экземпляр, построенный на Pack | 3.02 |
| Манифест | Назначение и границы Pack: зачем существует, что входит, что не входит | 3.03 |
| Сущность | Тип, онтология, отношение — элемент онтологии домена | 3.03 |
| Метод | Процесс, инструкция, протокол — способ действия в типовой ситуации | 3.03 |
| Онтологическая целостность | Состояние Pack, при котором понятия не дублируются, не противоречат, имеют связи с FPF | 3.04 |
| Meta-meta связь | Явная ссылка от понятия Pack к принципу FPF, SPF или ZP | 3.04 |
| Дрейф Pack | Расхождение между тем, что написано в Pack, и тем, что происходит в практике | 3.05 |
| Drift-detection | Процесс обнаружения дрейфа: скрипты и ручная проверка | 3.05 |
| SoT | Принцип разрешения дрейфа: решение, что обновлять — Pack или практику | 3.05 |
| Семейство документов | Группа родственных артефактов с общей структурой и критериями приёмки | 3.06 |
| Шаблон | Форма, задающая структуру без диктования содержания | 3.06 |
| ADR | Запись архитектурного решения с контекстом, альтернативами и последствиями | 3.07 |
| АрхГейт | Фильтр оценки архитектурных решений по семи характеристикам | 3.07 |

**Практика (15 мин):** Пройдитесь по списку. Для каждого понятия скажите вслух или запишите: «Это значит...» — своими словами, без чтения определения. Если застряли на понятии — вернитесь к соответствующему подразделу.
