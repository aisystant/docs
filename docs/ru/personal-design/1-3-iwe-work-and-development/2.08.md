#### §2.08 Список понятий раздела

Сводная таблица всех понятий, введённых в Разделе 2. Используйте её для проверки: можете ли вы объяснить каждое определение своими словами, не заглядывая в текст?

| Понятие | Краткое определение | Введено |
|---------|---------------------|---------|
| **Пилот** | Направляющая и решающая часть Болида: ценности, намерения, выбор направления, принятие решений в условиях неопределённости | §2.01 |
| **Машина** | Инфраструктурная часть Болида: устройства, репозитории, Pack, агенты, протоколы, резервные копии | §2.01 |
| **Болид** | Субъект, объединяющий Пилота и Машину в единую систему создания; не сумма, а новое качество с эмерджентными свойствами | §2.01 |
| **Pack** | Source-of-Truth IWE: структурированное хранилище форм, методов, категорий, принципов и состояния искусства | §2.02 |
| **DS** | Digital Self, производные от Pack: код, документация, заметки, рабочие продукты, созданные по правилам Pack | §2.02 |
| **SoT** | Принцип разрешения противоречий: если Pack и DS расходятся, правду несёт Pack, а DS подлежит обновлению | §2.02 |
| **Среда** | Инструментарий работы: рабочий стол, система версий, автоматизация, трекинг; не нейтральна, подталкивает к паттернам | §2.03 |
| **Агент** | Действующее лицо в IWE: модель ИИ + настройки + инструменты; работает в контуре, ограниченном MCP | §2.04 |
| **MCP** | Model Context Protocol, протокол связи между агентом и внешними системами; превращает агента из теоретика в практика | §2.04 |
| **Контур** | Граница, внутри которой агент может действовать; защищает от избыточных разрешений и размывания роли | §2.04 |
| **Аттестатор** | Функция Машины, фиксирующая поведенческие характеристики (bh) на основе событий: коммиты, задачи, время | §2.05 |
| **Диагност** | Функция Машины, оценивающая когнитивные параметры (cp) по запросу: картина мира, навыки, созидание | §2.05 |
| **Дрейф** | Расхождение между текущим состоянием Машины и намерениями пилота; неизбежен, но контролируем | §2.06 |
| **Интенция** | Направление, которое пилот задал Машине: методы, стандарты, цели | §2.06 |
| **Drift-detection** | Процесс обнаружения дрейфа: сверка Pack с практикой, проверка хуков, аудит агентов | §2.06 |
| **Дрейф-контроль** | Регулярное обнаружение и коррекция расхождения; три уровня: ежедневный, еженедельный, ежемесячный | §2.07 |
| **Аудит Машины** | Структурированная проверка по пяти критериям: наличие, актуальность, доступность, избыточность, документированность | §2.07 |
| **Онтологическая целостность** | Состояние, при котором все элементы Машины согласованы: Pack не противоречит DS, агенты не дублируют функции | §2.07 |
