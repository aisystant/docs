#### §8.06 Итерации и инкременты: стратегия растёт пошагово

**Время:** 45 мин чтение + 25 мин

**В одном предложении:** Стратегия выращивается через итерации: каждый цикл даёт инкремент — уточнение вектора, добавление домена или сдвиг вехи, — а микрорешения еженедельно уточняют один элемент каскада.

**Понятия этого раздела:**
- **Итерации** — Повторяемые циклы работы над стратегией или методом с проверкой результата. ≠ попытка, ≠ повтор
- **Инкременты** — Малые уточнения стратегии или метода, получаемые в результате итерации. ≠ изменение, ≠ улучшение
- **Микрорешения** — Малый итеративный шаг вместо крупного прыжка. ≠ задача, ≠ цель

**Мем, который снимается.** "Я сделаю идеальный каскад с первого раза." Стратегия выращивается, не создаётся. Каждая итерация даёт инкремент — уточнение вектора, добавление домена, сдвиг вехи. Микрорешения еженедельно уточняют один элемент на 10%.

**Определение из источника.**
**Итерация — цикл, не разовое действие.** Одно уточнение — инкремент. Серия уточнений через регулярные циклы — итерации.

**Инкремент — уточнение, не пересоздание.** Инкремент сохраняет основу и улучшает детали. Пересоздание с нуля — потеря накопленной ясности.

**Микрорешение — маленький шаг, не революция.** Уточнить вектор на 10% — микрорешение. Полностью переписать стратегию — революция, которая редко нужна.

**Развитие мысли.**
Пилоты часто пытаются создать идеальный каскад ВДВ с первого раза, тратят часы на формулировки, а потом не трогают документ месяцами. Стратегия не создаётся — она выращивается. Каждая итерация даёт уточнение, каждый инкремент — движение. Микрорешение в стратегировании: еженедельное уточнение одного элемента на 10%.

**Метод — минимальный шаг.**
1. Возьмите текущий каскад ВДВ. (~5 мин)
2. Выберите один элемент: вектор, домен или веху. (~10 мин)
3. Уточните его на 10%: более точная формулировка, конкретнее результат, добавьте измеримость. (~5 мин)
4. Зафиксируйте новую версию в Pack с датой. (~5 мин)

Взять текущий каскад ВДВ. Выбрать один элемент (вектор, домен или веху) и уточнить его на 10%. Зафиксировать новую версию в Pack с датой.

**Пример из жизни.** Аналитик тридцати одного года переписывал каскад ВДВ каждый месяц полностью — и терял накопленную ясность. Он перешёл на инкременты: каждую неделю уточнял один элемент на 10%. За 3 месяца вектор сузился с «развиваться» до «развить метод чтения для принятия решений» — без потери стабильности.

**Степени мастерства:**
| Степень | Что происходит | Критерий перехода |
|---------|----------------|-------------------|
| 1. Объяснение | Пытаюсь создать идеальную стратегию с первого раза | — |
| 2. Умение | Принимаю итеративный подход, но забываю обновлять | Один раз практически применил |
| 3. Навык | Регулярно вношу инкременты и фиксирую версии | Регулярно применяю без подсказки |
| 4. Мастерство | Управляю эволюцией стратегии через историю версий и системные итерации | Помогаю другим освоить, передаю культуру |

**Проверка себя.**
-  Я не пытаюсь сделать идеальный каскад с первого раза
-  У меня есть история версий каскада ВДВ
-  Я регулярно вношу микроинкременты, а не переписываю всё

**На практике.**
Через IWE Pack хранит историю версий каскада ВДВ, показывая эволюцию стратегии. Это позволяет видеть прогресс и не возвращаться к пройденному.
