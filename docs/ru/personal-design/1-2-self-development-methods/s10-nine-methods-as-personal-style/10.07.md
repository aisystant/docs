#### §10.07 Месячный аудит: замкнутый цикл развития

**Время:** 45 мин чтение + 25 мин

**В одном предложении:** Ежемесячный аудит 9 методов — статус, тренды, дрейф, новый bottleneck — замыкает цикл развития и обеспечивает стабильность системы методов через регулярную проверку и корректировку.

**Понятия этого раздела:**
- **Месячный аудит** — Регулярная ежемесячная проверка системы методов для предотвращения дрейфа. ≠ кризисная проверка, ≠ самооценка
- **Чеклист онтологической целостности** — Проверка соответствия между Pack, методами и практикой пилота. ≠ аудит, ≠ тест

**Мем, который снимается.** "Аудит — признак проблемы." Ежемесячный аудит — рутина стабильности, не признак кризиса. Без аудита система методов медленно распадается, пока bottleneck не блокирует всё.

**Определение из источника.**
**Аудит — проверка, не оценка.** Цель не «оценить себя», а собрать данные для корректировки.

**Тренд — динамика, не снимок.** Один аудит показывает состояние. Серия аудитов — тренд, который предсказывает будущее.

**Чеклист онтологической целостности — верификация.** Проверка: все понятия актуальны, связи корректны, bottleneck-подсказки работают, дублирования нет.

**Развитие мысли.**
Без регулярного аудита система методов медленно распадается: дрейф незаметен, bottleneck накапливает последствия, минимальная система размывается. Месячный аудит — это замкнутый цикл: проверить → скорректировать → сфокусироваться → практиковать → проверить снова. Каждый цикл укрепляет систему и готовит к переходу на следующую ступень.

**Метод — минимальный шаг.**
1. Проведите аудит 9 методов: статус, тренд, дрейф, bottleneck. (~5 мин)
2. Выберите 1 bottleneck-метод на следующий месяц. (~10 мин)
3. Зафиксируйте в Pack. (~5 мин)
4. Назначьте дату следующего аудита. (~5 мин)

Провести аудит 9 методов: статус, тренд, дрейф, bottleneck. Выбрать 1 bottleneck-метод на следующий месяц. Зафиксировать в Pack.

**Пример из жизни.** "Аудит — признак проблемы." Пилот проводил аудит 9 методов в первый раз. Дашборд показал: 3 метода на ступени «объяснение» (дрейф), 1 bottleneck (планирование), 1 разрыв (чтение без письма). Аудит не выявил кризис — он выявил точки роста. Следующий аудит запланирован через месяц.

**Степени мастерства:**
| Степень | Что происходит | Критерий перехода |
|---------|----------------|-------------------|
| 1. Объяснение | Не провожу аудит, действую наугад | — |
| 2. Умение | Провожу аудит время от времени | Один раз практически применил |
| 3. Навык | Системно провожу ежемесячный аудит и корректирую фокус | Регулярно применяю без подсказки |
| 4. Мастерство | Интегрирую аудит в замкнутый цикл через Диагност и Pack | Помогаю другим освоить, передаю культуру |

**Проверка себя.**
-  Я провожу ежемесячный аудит методов
-  Знаю свой bottleneck на текущий месяц
-  У меня есть запланированная дата следующего аудита

**На практике.**
Через IWE агент-Диагност генерирует автоматический отчёт «Методы» с рекомендациями: статус по каждому методу, тренды, дрейф, bottleneck на следующий месяц.

**См. также:** Дрейф методов — PD.GUIDE.2.S10.SS4, IWE усиление — PD.GUIDE.2.S10.SS6.

**Что дальше.** Этот раздел завершает описание девяти методов. Следующий шаг — применить их в ежедневной практике и пройти ежемесячный аудит через IWE.
