#### §8.08 Список понятий раздела

**Время:** 20 мин чтение + 40 мин = 60 мин
**Что узнаешь:** сводная таблица всех понятий Раздела 8 для самопроверки

---

Сводная таблица всех понятий, введённых в Разделе 8 «Системное моделирование». Используйте её для проверки: можете ли вы объяснить каждое определение своими словами, не заглядывая в текст?

| Понятие | Краткое определение | Введено |
|---------|---------------------|---------|
| **Черный ящик** | Описание системы через входы, выходы и функцию; модель, которая отвечает на вопрос «что она делает?» | §8.01 |
| **Вход** | То, что поступает в систему для преобразования; ресурс или стимул, который система обрабатывает | §8.01 |
| **Выход** | То, что система производит; результат преобразования входа | §8.01 |
| **Функция при черном ящике** | Желаемое действие системы, наблюдаемое снаружи; преобразование входа в выход ради цели, для которой система существует | §8.01 |
| **Прозрачный ящик** | Описание системы через внутреннее устройство: состав, процессы, механизмы; отвечает на «как устроена?» | §8.02 |
| **Внутреннее устройство** | Механизмы и процессы внутри границы системы, которые преобразуют входы в выходы | §8.02 |
| **Декомпозиция** | Разбиение прозрачного ящика на подсистемы для детального анализа с сохранением связи с функцией | §8.02 |
| **Функциональное описание** | Ответ на вопрос «что делает система?»; описание через входы, выходы и преобразования (черный ящик) | §8.03 |
| **Модульное описание** | Ответ на вопрос «из чего состоит система?»; описание через элементы, подсистемы и связи (прозрачный ящик) | §8.03 |
| **Пространственное описание** | Ответ на вопрос «где находится система?»; описание через геометрию, топологию, размещение элементов | §8.03 |
| **Стоимостное описание** | Ответ на вопрос «сколько стоит система?»; описание через ресурсы, затраты, ценность, экономику | §8.03 |
| **Модель** | Упрощённое описание системы, созданное для конкретной цели; не копия, а инструмент мышления | §8.04 |
| **Мета-модель** | Описание того, как строить модели; правила выбора, что включать в модель, а что исключать | §8.04 |
| **Мульти-модель** | Набор моделей одной системы, каждая из которых показывает свой аспект; четыре типа описания — пример | §8.04 |
| **Интегральная модель** | Попытка синтеза мульти-модели в единую картину; способ держать несколько моделей одновременно | §8.04 |
| **Верхнеуровневое моделирование** | Подход к построению моделей сверху вниз: от общей картины к деталям; сначала черный ящик, потом подсистемы | §8.05 |
| **Уровень детализации** | Степень подробности модели; глубина описания от «что делает» до «как устроен каждый винтик» | §8.05 |
| **Декомпозиция по уровням** | Последовательное разбиение системы на подсистемы с сохранением связи с верхним уровнем | §8.05 |
| **Интерфейс** | Совокупность правил, по которым система взаимодействует с другими системами; граница обмена | §8.06 |
| **Интерфейсная спецификация** | Формальное описание интерфейса: форматы данных, протоколы, ограничения; контракт между системами | §8.06 |
| **Скрытость реализации** | Принцип, при котором внутреннее устройство скрыто за интерфейсом; позволяет менять устройство без нарушения связей | §8.06 |
| **Характеристика** | Потенциальное свойство системы; то, что система способна делать в принципе, независимо от текущего момента | §8.07 |
| **Состояние** | Моментное значение характеристики в конкретный момент времени; то, что система демонстрирует «сейчас» | §8.07 |
| **Показатель** | Способ измерения характеристики; метрика, которая превращает качественное описание в количественное наблюдение | §8.07 |

---

**Практика — самопроверка «Объясни своими словами» (40 мин):**

1. Закройте текст подразделов §8.01–§8.07 (5 мин).
2. Пройдитесь по таблице сверху вниз. Для каждого понятия произнесите вслух или запишите: своими словами, что это значит? Приведите пример из своей жизни (25 мин).
3. Откройте соответствующий подраздел и проверьте: ваше объяснение совпадает по смыслу с определением? Где расхождения — там пробел в понимании (10 мин).
