#### §5.09 Упражнения на понятия

Три практических упражнения для закрепления понятий раздела 5.

---

**Упражнение 1. Карта предметов интереса (20 мин)**

Выберите систему, с которой вы работаете прямо сейчас (продукт, сервис, процесс, команду).

Заполните таблицу для трёх ролей:

| Роль | Предмет интереса | Ролевой интерес | Предпочтение |
|------|-----------------|-----------------|--------------|
| Роль 1 | | | |
| Роль 2 | | | |
| Роль 3 | | | |

После заполнения: найдите конфликт предпочтений. Как бы вы проектировали систему, чтобы удовлетворить все три ролевых интереса?

---

**Упражнение 2. Потребности — Требования — Ограничения (25 мин)**

Возьмите реальный запрос или задачу, которую вы сейчас решаете.

1. Запишите всё, что сказано о том, что нужно от системы или решения — без сортировки.
2. Разложите по трём колонкам: потребности (зачем), требования (что именно), ограничения (что нельзя изменить).
3. Для каждого требования задайте вопрос: какую потребность оно обслуживает? Есть ли требования без явной потребности за ними?

Вывод: напишите потребность в одном предложении («система нужна, чтобы…»).

---

**Упражнение 3. Система-создатель: ролевой аудит (15 мин)**

Нарисуйте схему «система-создатель» для вашего текущего проекта.

1. В центре — целевая система (что создаётся).
2. Вокруг — организационные звенья (кто участвует в создании).
3. Для каждого звена — организационная роль (функция звена в создании).
4. Отметьте: есть ли нужные роли, которые не исполняются ни одним звеном?
