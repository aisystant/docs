# Язык инженера и менеджера: различение по семантике

Операционный менеджер и инженер смотрят на один и тот же кусок реальности, но из разных **viewpoint**.

* Инженер чаще описывает **метод**: «как сделать правильно».
* Операционный менеджер чаще описывает **работу в потоке**: «кто, когда, с каким ресурсом и с каким риском сделает».

Одна и та же фраза может иметь разный смысл из разных viewpoint.

Пример: «Нужно повысить качество».

* В инженерном viewpoint это часто про метод, архитектуру, покрытие тестами.
* В операционном viewpoint это может быть про стабильность потока, снижение переделок, предсказуемость сроков.

Без явного указания viewpoint беседа быстро превращается в спор, где каждый «прав внутри своей рамки».
